# Project Architecture & Patterns Summary

## 🏗️ Monorepo Structure
The project is a TypeScript monorepo managed with **pnpm workspaces**. It is organized into several key packages:
- **`packages/core`**: The central service (Hub) built with Node.js, Fastify, and Socket.io. It handles core logic, agent execution, and manages various subsystems via file-watching "Managers".
- **`packages/ui`**: A shared React component library used by the web dashboard and potentially other interfaces. It uses Tailwind CSS and Lucide icons.
- **`apps/web`**: A Next.js application that serves as the primary Web UI/Dashboard.
- **`packages/cli`**: A command-line interface (`super-ai`) for controlling the Hub.
- **`packages/vscode`**: A VS Code extension that connects to the Hub.
- **`packages/browser`**: A Chrome extension for browser integration and memory capture.
- **`packages/adapters`**: Wrapper scripts for different LLM providers (Gemini, Claude) and hook emission.

## ⚙️ Core Architecture & Patterns
### Manager Pattern
The `core` package utilizes a highly modular "Manager" pattern. Each manager is responsible for a specific domain and often uses `chokidar` for recursive directory watching to support dynamic updates:
- `AgentManager`, `SkillManager`, `HookManager`, `PromptManager`, `ContextManager`, `CommandManager`, `SchedulerManager`, `MemoryManager`, `ProfileManager`, `DocumentManager`, `HandoffManager`, `SessionManager`, `SystemPromptManager`.

### MCP (Model Context Protocol)
MCP is a first-class citizen in this architecture. The project acts as both:
- **MCP Client**: Aggregates multiple MCP servers, handles lifecycle management, and provides semantic tool search.
- **MCP Server**: Can act as a "master" server to expose internal tools to other AI assistants.
- **`McpProxyManager`**: Manages the proxying and routing of MCP traffic, including TOON format conversion.

### UI & UX Patterns
- **Responsiveness**: Standardized use of `useResizeObserver` (recently refactored into a shared hook) for canvas-based components like `KnowledgeGraph` and `GraphPanel`.
- **Dynamic Imports**: Components using browser-only APIs (like `react-force-graph-2d`) are dynamically imported with `ssr: false` to ensure Next.js compatibility.
- **Theme Support**: Integrated with `next-themes` for seamless light/dark mode transitions.
- **Data Fetching**: Uses **tRPC** for type-safe communication between the frontend and the backend Hub.

## 🤖 AI & Agent Orchestration
- **`AgentExecutor`**: The main engine for running agents, abstracting LLM providers via `ModelGateway`.
- **`ModelGateway`**: Provides a unified interface for OpenAI, Anthropic, and Ollama.
- **Multi-Agent Systems**: Supports subagents, council of supervisors, and consensus/debate protocols.
- **`TrafficObserver`**: Implements "passive memory" by extracting facts from interactions and storing them in `MemoryManager`.

## 📍 Key Decisions & Roadmap Goals
- **Autopilot & Autonomous Operation**: Aiming for a fully autonomous development experience where the "Director" or "Council" can steer tasks without constant user approval.
- **Feature Parity**: Driving towards parity with major AI coding CLIs (Claude Code, Opencode, Codebuff) and specialized forks (metamcp, jules-autopilot).
- **Universal Memory**: Implementing a comprehensive RAG and memory system that spans short-term session context and long-term knowledge storage.
- **Unified Dashboard**: A single interface to manage prompts, skills, MCP servers, and agent sessions across local and cloud environments.

## 🛠️ Tech Stack Highlights
- **Backend**: Node.js, Fastify, Socket.io, tRPC, Fuse.js (fuzzy search), node-schedule.
- **Frontend**: React, Next.js, Tailwind CSS, Framer Motion, ForceGraph2D.
- **Languages**: TypeScript (monorepo-wide), Go (referenced for future porting/efficiency).
